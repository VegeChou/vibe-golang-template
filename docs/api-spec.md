# API Spec Entry

The primary specification has been moved to an OpenAPI rules file for LLM and tooling consumption:

- [../rules/unified-api.openapi.yaml](../rules/unified-api.openapi.yaml)
- [../rules/unified-api.human.md](../rules/unified-api.human.md)

This includes:

- Unified response envelope (`ApiResponse`)
- Language negotiation (`Accept-Language` / `lang`, supports `zh-CN` and `en-US`)
- Pagination schemas (`PageResult`, `CursorResult`)
- Error codes and HTTP mapping (`ErrorCode`)
- Error response and `traceId` conventions

Sync policy:

- OpenAPI is the only manually edited source: `rules/unified-api.openapi.yaml`
- The human-readable doc is generated by script: `bash scripts/sync-api-docs.sh`
- CI verifies generated output stays in sync: `.github/workflows/verify-api-doc-sync.yml`
- Run [../scripts/install-githooks.sh](../scripts/install-githooks.sh) to install local hooks (auto-sync on commit)
